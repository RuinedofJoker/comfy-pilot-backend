# ComfyUI Pilot 需求文档

## 文档结构

### 核心文档
- **00-项目概述.md**: 项目定位、核心价值、系统架构、技术栈
- **01-核心数据模型.md**: 所有数据表设计、字段定义、索引、关系图
- **02-API设计.md**: 所有API接口定义、请求响应格式

### 模块设计文档
- **03-用户模块.md**: 用户认证、权限管理
- **04-服务模块.md**: ComfyUI服务管理
- **05-工作流模块.md**: 工作流CRUD、版本管理
- **06-会话模块.md**: AI对话、消息管理
- **07-Agent模块.md**: Agent配置、调度、执行
- **08-AI模型模块.md**: 模型提供商、模型配置、调用管理
- **09-资源模块.md**: 文件上传、存储管理

### 技术规范
- **99-技术规范.md**: 架构规范、代码规范、数据库规范、API规范、安全规范

## 文档特点

### 简洁性
- 去除冗余内容
- 保留核心信息
- 便于快速查阅

### 结构化
- 按模块组织
- 清晰的层次结构
- 易于维护更新

### 实用性
- 面向代码实现
- 包含关键技术细节
- 支持快速开发

## 使用建议

### 开发流程
1. 阅读 **00-项目概述.md** 了解整体架构
2. 查看 **01-核心数据模型.md** 理解数据结构
3. 参考 **02-API设计.md** 了解接口定义
4. 根据具体模块查看对应的模块设计文档
5. 遵循 **99-技术规范.md** 进行编码

### 文档维护
- 新增功能时更新对应模块文档
- 数据表变更时更新核心数据模型文档
- API变更时更新API设计文档
- 保持文档与代码同步
