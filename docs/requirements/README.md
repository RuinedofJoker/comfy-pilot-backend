# ComfyUI Pilot 需求文档

## 文档结构

### 核心文档
- **00-项目概述.md**: 项目定位、核心价值、系统架构、技术栈
- **99-技术规范.md**: 架构规范、代码规范、数据库规范、API规范、安全规范

### 模块文档说明
每个模块在独立子目录下包含三个文件：
- **模块设计.md**: 模块功能、依赖关系、业务逻辑
- **数据模型.md**: 数据表设计、字段定义、索引
- **API设计.md**: 接口定义、请求响应格式

### 业务核心模块
- **03-用户模块/**: 用户信息管理
- **04-认证模块/**: 登录认证、Token管理、密码重置
- **05-权限模块/**: 角色管理、权限控制、访问校验
- **06-ComfyUI服务模块/**: ComfyUI服务配置、连接测试、状态查询
- **07-工作流模块/**: 工作流CRUD、版本管理、快照机制、锁定控制
- **08-会话模块/**: AI对话、消息管理、WebSocket通信、工作流同步
- **09-Agent模块/**: Agent配置、调度、执行
- **10-AI模型模块/**: 模型提供商、模型配置、接口调用
- **11-资源模块/**: 文件上传、存储管理

### 支撑服务模块
- **12-通知模块/**: 邮件发送、模板管理、异步队列
- **13-监控模块/**: 服务健康检查、状态管理、统计数据
- **14-审计日志模块/**: 操作记录、安全审计、日志查询
- **15-Tool模块/**: 工具抽象层、工具注册、工具执行

## 文档特点

### 简洁性
- 去除冗余内容
- 保留核心信息
- 便于快速查阅

### 结构化
- 按模块组织
- 清晰的层次结构
- 易于维护更新

### 实用性
- 面向代码实现
- 包含关键技术细节
- 支持快速开发

## 使用建议

### 开发流程
1. 阅读 **00-项目概述.md** 了解整体架构
2. 查看每个模块自己的核心数据模型.md理解数据结构
3. 参考每个模块自己的API设计.md了解接口定义
4. 根据具体模块查看对应的模块设计文档
5. 遵循 **99-技术规范.md** 进行编码

### 文档维护
- 新增功能时更新对应模块文档
- 数据表变更时更新核心数据模型文档
- API变更时更新API设计文档
- 保持文档与代码同步
